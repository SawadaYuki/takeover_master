00060 /* Linux uses a negative return value to indicate syscall errors,
00061    unlike most Unices, which use the condition codes' carry flag.
00062 
00063    Since version 2.1 the return value of a system call might be
00064    negative even if the call succeeded.  E.g., the `lseek' system call
00065    might return a large offset.  Therefore we must not anymore test
00066    for < 0, but test for a real error by making sure the value in %eax
00067    is a real error number.  Linus said he will make sure the no syscall
00068    returns a value in -1 .. -4095 as a valid result so we can safely
00069    test with -4095.  */

00173 /* The Linux/x86-64 kernel expects the system call parameters in
00174    registers according to the following table:
00175 
00176     syscall number      rax
00177     arg 1               rdi
00178     arg 2               rsi
00179     arg 3               rdx
00180     arg 4               r10
00181     arg 5               r8
00182     arg 6               r9
00183 
00184     The Linux kernel uses and destroys internally these registers:
00185     return address from
00186     syscall             rcx
00187     additionally clobbered: r12-r15,rbx,rbp
00188     eflags from syscall r11
00189 
00190     Normal function call, including calls to the system call stub
00191     functions in the libc, get the first six parameters passed in
00192     registers and the seventh parameter and later on the stack.  The
00193     register use is as follows:
00194 
00195      system call number in the DO_CALL macro
00196      arg 1              rdi
00197      arg 2              rsi
00198      arg 3              rdx
00199      arg 4              rcx
00200      arg 5              r8
00201      arg 6              r9
00202 
00203     We have to take care that the stack is aligned to 16 bytes.  When
00204     called the stack is not aligned since the return address has just
00205     been pushed.
00206 
00207 
00208     Syscalls of more than 6 arguments are not supported.  */
00209
