00052 /* Linux uses a negative return value to indicate syscall errors,
00053    unlike most Unices, which use the condition codes' carry flag.
00054 
00055    Since version 2.1 the return value of a system call might be
00056    negative even if the call succeeded.  E.g., the `lseek' system call
00057    might return a large offset.  Therefore we must not anymore test
00058    for < 0, but test for a real error by making sure the value in %eax
00059    is a real error number.  Linus said he will make sure the no syscall
00060    returns a value in -1 .. -4095 as a valid result so we can safely
00061    test with -4095.  */

00202 /* Linux takes system call arguments in registers:
00203 
00204         syscall number  %eax         call-clobbered
00205         arg 1           %ebx         call-saved
00206         arg 2           %ecx         call-clobbered
00207         arg 3           %edx         call-clobbered
00208         arg 4           %esi         call-saved
00209         arg 5           %edi         call-saved
00210 
00211    The stack layout upon entering the function is:
00212 
00213         20(%esp)        Arg# 5
00214         16(%esp)        Arg# 4
00215         12(%esp)        Arg# 3
00216          8(%esp)        Arg# 2
00217          4(%esp)        Arg# 1
00218           (%esp)        Return address
00219 
00220    (Of course a function with say 3 arguments does not have entries for
00221    arguments 4 and 5.)
00222 
00223    The following code tries hard to be optimal.  A general assumption
00224    (which is true according to the data books I have) is that
00225 
00226         2 * xchg        is more expensive than  pushl + movl + popl
00227 
00228    Beside this a neat trick is used.  The calling conventions for Linux
00229    tell that among the registers used for parameters %ecx and %edx need
00230    not be saved.  Beside this we may clobber this registers even when
00231    they are not used for parameter passing.
00232 
00233    As a result one can see below that we save the content of the %ebx
00234    register in the %edx register when we have less than 3 arguments
00235    (2 * movl is less expensive than pushl + popl).
00236 
00237    Second unlike for the other registers we don't save the content of
00238    %ecx and %edx when we have more than 1 and 2 registers resp.
00239 
00240    The code below might look a bit long but we have to take care for
00241    the pipelined processors (i586).  Here the `pushl' and `popl'
00242    instructions are marked as NP (not pairable) but the exception is
00243    two consecutive of these instruction.  This gives no penalty on
00244    other processors though.  */

00248     PUSHARGS_##args                                                           \
00249     DOARGS_##args                                                             \
00250     movl $SYS_ify (syscall_name), %eax;                                       \
00251     ENTER_KERNEL                                                              \
00252     POPARGS_##args
